# Контекст проекта

## Текущее состояние
- Приложение: Todo List
- Архитектура: Clean Architecture
- Состояние: В разработке
- Основные технологии: Flutter, BLoC, Hive

## Ключевые требования
1. Функциональные:
   - Создание задач
   - Редактирование задач
   - Удаление задач
   - Отметка о выполнении
   - Локальное хранение

2. Нефункциональные:
   - Тестируемость
   - Масштабируемость
   - Производительность
   - Безопасность

## Технический стек
- Flutter 3.x
- BLoC для state management
- Hive для локального хранения
- Mockito для тестирования
- GetIt для DI

## Архитектурные ограничения
1. Слои:
   - Domain (бизнес-логика)
   - Data (репозитории)
   - Presentation (UI)
   - Core (утилиты)

2. Зависимости:
   - Presentation → Domain
   - Data → Domain
   - Core может использоваться везде
